Why xxtui

为什么用 xxtui 来做个人推送

自己搭 webhook、邮件服务太麻烦？xxtui 帮你把这些重复工作「打包好」。 你只关注要发什么内容、什么时候发，其它交给平台。

接入极简 · 一条 URL 搞定

不需要复杂 SDK，也不用注册一堆第三方应用。 登录获取 API_KEY，填进 URL，立刻开始推送。 适合脚本、定时任务、Serverless 函数等各种环境。

能发 HTTP 请求的地方，就能用 xxtui。

多渠道合一 · 一次请求多端到达

同一条消息支持同时发送到 微信公众号、企业微信群机器人、钉钉群机器人、飞书机器人、Bark、短信 / 语音、邮件以及你的自定义 HTTP 接口。

不再重复接入每个平台的通知能力。

为个人设计 · 够轻量也够可控

适合个人用户与小团队自用：有额度限制防止误发炸号，也提供消息记录、渠道配置等能力，方便排查和迭代。

日常「折腾」和生产通知，都能逐步迁移过来。

Getting Started

3 步完成接入

正常情况下，从「第一次打开 xxtui」到发出第一条推送，只需要几分钟。

STEP 01
注册 / 登录 xxtui

访问 xxtui.com， 使用常用账号登录。进入控制台后，在「账号管理」中查看你的 API_KEY。

STEP 02
配置推送渠道

根据需要绑定微信公众号、企业微信群机器人、钉钉、飞书、Bark、邮箱、手机等。 也可以配置自定义 HTTP 接口，扩展到你自己的服务。

STEP 03
在代码中发起请求

把示例 URL 中的 <API_KEY> 替换为你的 key， 写入脚本、任务或监控系统中。以后所有通知，直接通过 xxtui 统一下发。

API Example

常用调用方式示例

下面是 curl 与 Python 的调用示例。 请根据官网文档实际参数为准，你可以根据自身需要加上 from、 title、channel 等参数。

curl -X POST "https://www.xxtui.com/xxtui/<API_KEY>" \ -H "Content-Type: application/json" \ -d '{ "content": "备份完成 ✅", "title": "服务器通知", "from": "prod-bak" }'
import requests API_KEY = "<API_KEY>" url = f"https://www.xxtui.com/xxtui/{API_KEY}" payload = { "content": "任务执行完毕 ✅", "title": "定时任务通知", "from": "python-cron", # "channel": "WX_MP,EMAIL", # 可选 } resp = requests.post(url, json=payload, timeout=5) print(resp.status_code, resp.text)
Use Cases

典型使用场景

xxtui 适合所有「我只想简单收个通知」的场景，无论你是开发者、运维工程师，还是只是想给自己的生活加点自动化。

个人运维 / 折腾党

  • 服务器 CPU / 磁盘 / 内存告警，通过企业微信或钉钉推送。
  • 定时备份结果通知，失败立刻收到消息。
  • 自己写的小服务有异常，自动发消息到 Bark 或邮箱。

生活提醒 / 个人效率

  • 信用卡还款、电费水费、定期任务等通过脚本推送提醒。
  • RSS / 抓取脚本有最新内容，自动转发到个人渠道。
  • 把零散通知统一推到一个「消息收件箱」渠道里。

小团队 / Side Project

  • 团队共享的「故障通知群」，由 xxtui 聚合不同服务的告警。
  • 为 side project 快速加上邮件 / 机器人通知，不用额外搭建。
  • 先用 xxtui 验证通知流程，后续再考虑自建服务也不迟。
